# Техническое описание приложения "МКД-Менеджер"

## 1. Общие сведения

**Название приложения:** МКД-Менеджер (Многоквартирный Дом - Менеджер)  
**Назначение:** Система автоматизации планирования, структурирования и контроля работ по содержанию и ремонту многоквартирных домов  
**Тип приложения:** Настольное приложение на Python с графическим интерфейсом  
**Архитектурный стиль:** Многослойное приложение с разделением на представление, бизнес-логику и слой данных

## 2. Архитектура приложения

### 2.1 Общая архитектура

Приложение построено по многослойной архитектуре:

- **Слой представления (GUI):** `/mkd_manager/gui/` - реализует пользовательский интерфейс
- **Слой бизнес-логики:** `/mkd_manager/core/`, `/mkd_manager/rules/` - содержит основную логику приложения
- **Слой данных:** `/mkd_manager/models/`, `/mkd_manager/database/` - работа с данными
- **Слой конфигурации:** `/mkd_manager/config/` - настройки приложения
- **Вспомогательные модули:** `/mkd_manager/reports/` - генерация отчетов

### 2.2 Зависимости

Основные зависимости приложения определены в `/mkd_manager/requirements.txt`:
- `sqlmodel` - ORM для работы с базой данных
- `pyqt6` - фреймворк для графического интерфейса
- `networkx` - для построения графов зависимостей
- `matplotlib` - визуализация графов
- `pyyaml` - работа с конфигурационными файлами
- `jinja2` - шаблонизация
- `reportlab` - генерация PDF-отчетов

## 3. Подробное описание модулей

### 3.1 Модуль конфигурации (`/mkd_manager/config/`)

#### 3.1.1 `settings.py`
Класс `Settings` загружает конфигурацию из файла `settings.yaml` и предоставляет доступ к настройкам приложения. Содержит следующие секции:
- `app` - общие параметры приложения (название, версия, режим отладки)
- `database` - параметры подключения к базе данных
- `paths` - пути к различным ресурсам приложения
- `defaults` - значения по умолчанию для различных параметров

#### 3.1.2 `settings.yaml`
Файл конфигурации в формате YAML, содержащий:
- Настройки подключения к SQLite базе данных
- Пути к файлам и директориям
- Параметры по умолчанию для работы с МКД

### 3.2 Модель данных (`/mkd_manager/models/`)

#### 3.2.1 `building.py`
Класс `Building` представляет собой модель данных для многоквартирного дома. Содержит:
- Паспортные данные дома (кадастровый номер, адрес, год постройки, этажность и т.д.)
- Характеристики инженерных систем (наличие лифтов, мусоропровода, газа и т.д.)
- Показатели износа конструктивных элементов
- Связь с работами через `BuildingWork`

#### 3.2.2 `work.py`
Классы модели работ:
- `WorkCategory` - перечисление категорий работ (содержание, ремонт, управление и др.)
- `WorkType` - перечисление типов работ (плановые, аварийные, по заявке)
- `Work` - модель конкретной работы с полным описанием (код, название, периодичность, нормативы и т.д.)
- `WorkDependency` - модель зависимостей между работами
- `BuildingWork` - промежуточная модель связи работ с конкретными домами

### 3.3 База данных (`/mkd_manager/database/`)

#### 3.3.1 `init_db.py`
Скрипт инициализации базы данных:
- Создает директорию для базы данных если она не существует
- Инициализирует движок SQLAlchemy
- Создает все таблицы в базе данных
- Загружает начальные данные работ из JSON файла
- Проверяет наличие данных перед повторной загрузкой

#### 3.3.2 `mkd.db`
Файловая SQLite база данных, содержащая:
- Таблица `buildings` - информация о домах
- Таблица `works` - справочник работ
- Таблица `building_works` - связи между домами и работами
- Таблица `work_dependencies` - зависимости между работами

### 3.4 Бизнес-логика (`/mkd_manager/core/`)

#### 3.4.1 `graph_engine.py`
Модуль построения и управления графами работ:
- Класс `WorkGraph` реализует направленный граф зависимостей
- Методы добавления работ и зависимостей
- Алгоритмы определения порядка выполнения работ
- Поиск критического пути
- Визуализация графа зависимостей
- Поддержка разных типов зависимостей (порядковые, ресурсные, логические)

### 3.5 Правила применимости (`/mkd_manager/rules/`)

#### 3.5.1 `applicability_rules.py`
Содержит бизнес-правила определения применимости работ к конкретному дому:
- Проверка по категории работ (лифты, газ, мусоропровод и т.д.)
- Проверка по износу элементов здания
- Проверка требований ОСС (общего собрания собственников)
- Расчет объемов работ в зависимости от характеристик дома
- Логика принятия решений о применимости работ

### 3.6 Графический интерфейс (`/mkd_manager/gui/`)

#### 3.6.1 `main_window.py`
Реализация главного окна приложения:
- Главное меню и панель инструментов
- Вкладки с различными разделами (МКД, Работы, План работ, Отчеты)
- Таблицы для отображения данных
- Формы для редактирования информации
- Диалоги подтверждения и сообщений
- Интеграция с бизнес-логикой приложения

### 3.7 Данные приложения (`/mkd_manager/data/`)

#### 3.7.1 `works_full.json`
JSON файл с полным перечнем работ по содержанию и ремонту МКД:
- Более 270 видов работ
- Классификация по ЖНМК 2.x.x
- Параметры работ (периодичность, нормативы, единицы измерения)
- Минимальные показатели износа
- Признаки критичности

### 3.8 Отчеты (`/mkd_manager/reports/`)

#### 3.8.1 Структура модуля
- Директория `templates/` для хранения шаблонов отчетов
- Модуль для генерации отчетов (в разработке)
- Поддержка различных форматов вывода (PDF, Excel)

### 3.9 Точка входа (`/mkd_manager/main.py`)

Основной файл запуска приложения:
- Инициализация основного интерфейса
- Запуск графического интерфейса
- Обработка начальных параметров запуска

## 4. Функциональные возможности

### 4.1 Управление домами
- Ввод и редактирование паспортных данных домов
- Учет характеристик инженерных систем
- Ведение показателей износа

### 4.2 Управление работами
- Классификация работ по категориям
- Учет нормативной базы
- Планирование периодичности выполнения
- Расчет объемов работ

### 4.3 Планирование работ
- Формирование планов работ по домам
- Учет зависимостей между работами
- Построение графа работ
- Определение критического пути

### 4.4 Контроль выполнения
- Отслеживание статуса работ
- Сравнение плановых и фактических данных
- Анализ выполнения планов

### 4.5 Генерация отчетов
- Формирование отчетов по выполненным работам
- Статистика по домам и работам
- Финансовая отчетность

## 5. Технологии и подходы

### 5.1 Примененные технологии
- **Python 3.12** - основной язык программирования
- **SQLModel** - ORM с поддержкой Pydantic
- **PyQt6** - графический интерфейс
- **NetworkX** - алгоритмы работы с графами
- **SQLite** - файловая база данных
- **YAML** - конфигурационные файлы

### 5.2 Архитектурные паттерны
- **Разделение ответственности** - каждый модуль выполняет свою функцию
- **Слабая связанность** - модули взаимодействуют через четко определенные интерфейсы
- **Инверсия зависимостей** - высокоуровневые модули не зависят от низкоуровневых

### 5.3 Принципы проектирования
- **SOLID** - принципы объектно-ориентированного проектирования
- **DRY** - Don't Repeat Yourself
- **KISS** - Keep It Simple, Stupid
- **YAGNI** - You Aren't Gonna Need It
